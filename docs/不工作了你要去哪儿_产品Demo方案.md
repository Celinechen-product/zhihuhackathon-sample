# 《人生样本库》产品 Demo 方案｜当前联调版

> 当前 Demo 文件：`not-working-new-zealand-demo.html`  
> 当前产品名：**人生样本库**  
> 当前示例主问题：**不工作了之后，我想去新西兰生活**  
> 当前后端地址：`http://127.0.0.1:8000`  
> 前端本地服务建议地址：`http://127.0.0.1:5173/`

本方案记录当前前端 Demo 的最新产品链路、页面结构、交互方式和后端联调边界。旧版中的拓扑图、全屏提问弹窗、单一问题样本库、示例 chips、黑色主按钮等设计均已废弃。

---

## 1. 产品定位

### 1.1 产品名

```text
人生样本库
```

### 1.2 一句话

用户写下一个真实人生困惑，系统从知乎公开经历中找到走过相似道路的人，让用户看见他们怎么做、读到他们写过的内容，并以低打扰的方式继续提问。

### 1.3 产品边界

- 不直接给标准答案；
- 不替用户做人生决策；
- 不伪装成真人即时回复；
- 不默认真实发送私信或真实发布知乎内容；
- Demo 阶段允许 mock fallback，但真实接口位置已保留。

## 2. 登录与后端联调

### 2.1 前置知乎授权登录

用户访问产品网页时，前端先检查 session 登录状态。

```text
访问前端页面
→ GET http://127.0.0.1:8000/auth/me
→ 若未登录，跳转 http://127.0.0.1:8000/auth/zhihu/login
→ 后端完成知乎 OAuth
→ 后端写入 session cookie
→ 回跳 http://127.0.0.1:5173/
→ 前端再次请求 /auth/me 判断登录态
```

登录成功后：

- 右上角显示用户知乎头像；
- hover 头像显示知乎名称；
- 点击头像，新窗口打开知乎个人主页。

### 2.2 当前搜索接口消费

默认搜索接口：

```text
GET http://127.0.0.1:8000/api/search?query={query}&count=20
```

兼容旧 Demo 接口：

```text
POST http://127.0.0.1:8000/api/demo/search
```

只有设置 `window.LIFE_PATH_USE_DEMO_SEARCH = true` 时才走旧 Demo 接口。

### 2.3 AI 分身聊天接口

```text
POST http://127.0.0.1:8000/api/personas/chat
```

用于 AI 分身聊天页。请求失败时前端会使用本地 mock 回复兜底。

## 3. 首页

### 3.1 首页主结构

首页是一个居中的输入产品，不再是营销页。

首屏内容：

```text
有什么让你纠结了很久的事？越具体越好

[大输入框]
```

首页只保留：

- 顶部导航；
- 产品名和小人 Logo；
- 一行主标题；
- 一个大输入框；
- 输入框右下角发送箭头。

已删除：

- `知乎出品 · 找到走过相似道路的人`
- `写下一个真实困惑...`
- `寻找走过的人` 文字 CTA
- 示例问题 chips
- 首页底部文案

### 3.2 输入框底纹词

输入框内轮换展示底纹词：

```text
不工作了之后，我想去新西兰生活
为了工作，异地恋值得吗？
有什么值得白领尝试的体力劳动
30岁裸辞后怎么办
```

交互：

- 如果用户未输入，点击发送箭头时，当前轮换到底纹词作为 query；
- 如果用户点击输入框，底纹词消失，用户可自行输入；
- 用户点击发送并真正开始搜索时，记录：

```js
state.hasSubmittedQuestion = true;
state.submittedQuestion = state.question;
```

该记录用于后续「我的样本库」按问题归档。

### 3.3 首页视觉

- 使用 Intercom 暖奶油色视觉；
- 首页主内容用 flex 垂直居中；
- 标题和输入框整体位于视口视觉中心附近；
- 输入框高度较高，底纹词轻、细、不抢主操作。

## 5. 前人路径页

### 5.1 页面标题

```text
关于「{用户输入的问题}」，TA 们是这样做的...
```

### 5.2 路径卡橱窗

页面初始展示 5 张路径卡，不恢复拓扑图、放射路线或关系图。

路径数据：

1. 先去一年试生活｜128 人走过
2. 留学转工签｜76 人走过
3. 去了又回来｜61 人走过
4. 技术迁移留下来｜54 人走过
5. 远程工作旅居｜43 人走过

每张路径卡只展示：

- 序号 + track name；
- `xx 人走过`；
- 一句话解释；
- 底部最多 4 个前人头像。

不展示：

- `适合你看...`
- hover 常见问题；
- 页面级底部出口；
- 拓扑图。

### 5.3 路径卡内部排版

路径卡内部结构：

```html
<button class="path-card">
  <div class="path-card-header">
    <span class="path-index">01</span>
    <h2>先去一年试生活</h2>
  </div>
  <span class="path-count">128 人走过</span>
  <p class="path-desc">路径解释</p>
  <span class="path-card-peeps">头像</span>
</button>
```

排版规则：

- 序号 badge 和路径名第一行顶部对齐；
- 标题区固定高度，保证 5 张卡的 `xx 人走过` 顶部统一；
- 路径名到人数间距约 18px；
- 人数到解释间距约 12px；
- 底部头像固定在底部，不参与文字布局。

### 5.4 路径点击后的展开

点击某张路径卡后，不跳转新页面。

当前交互：

```text
初始状态：只展示 5 张路径卡
点击某路径卡
→ 在 5 张路径卡下方出现该路径的人物样本库
→ 页面自动滚动到下方样本库
→ 用户上滑仍能看到上方 5 张路径卡
→ 点击另一张路径卡，下方样本库内容替换为新路径
```

不再使用：

- 独立 `samples` 页面作为路径主入口；
- `收起，看看其他路径` 按钮；
- 点击后整页闪跳。

展开区标题：

```text
{路径名}的人
```

副标题：

```text
该路径的一句话解释
```

## 6. 路径内前人样本库

### 6.1 人物舞台

路径展开后，展示横向人物样本舞台。

要求：

- 人物头像来自后端检索返回的真实用户头像；
- 若后端字段缺失，前端使用 mock fallback；
- 中央人物可出现气泡；
- 点击头像后打开该人物样本卡；
- 其他人物弱化或收起，保持舞台空间。

### 6.2 人物样本卡

人物样本卡信息顺序：

1. TA 是谁
2. 为什么匹配到 TA
3. TA 的经历时间线
4. TA 的真实经历
5. 底部操作按钮

人物名称：

- 使用后端返回的知乎昵称；
- 可点击；
- 新窗口打开对应知乎主页。

已删除：

- `前人样本｜路径名`
- `TA 留下的关键经验`
- `再看看其他人`
- 样本卡内的知乎圈子出口。

底部按钮：

```text
向 TA 提问
存入我的样本库
```

### 6.3 TA 的真实经历

原 `TA 写过的原文入口` 改为：

```text
TA的真实经历
```

原文入口不做完整卡片，只保留文本与暖黄色虚线下划线，表示可点击。

## 7. 阅读原文与提问面板

### 7.1 阅读原文

点击某篇真实经历后，使用右侧沉浸式展开。

布局：

```text
左侧：人物样本卡收窄
右侧：原文阅读器大面板
```

不跳转新页面，不出现全屏弹窗。

### 7.2 向 TA 提问

点击人物样本卡里的：

```text
向 TA 提问
```

不再打开独立页面，也不再打开全屏弹窗。

当前交互：

```text
人物样本卡左移并收窄
右侧展开提问面板
右侧面板和阅读原文共用沉浸式布局
```

右侧提问面板内容：

- 顶部弱提示：`向 TA 提问`
- 一个大输入框；
- 底部按钮：

```text
呼叫 TA 的 AI 分身先聊聊
递给 TA 本人
先不问，我再看看
```

其中：

- `呼叫 TA 的 AI 分身先聊聊`：读取输入框内容，进入 AI 分身聊天页；
- `递给 TA 本人`：Demo 阶段只 toast，不真实发送；
- `先不问，我再看看`：弱文字按钮，收起右侧提问面板。

右侧区域互斥：

```text
readerOpen 和 askPanelOpen 不会同时打开
```

## 8. AI 分身聊天页

入口：

- 从右侧提问面板点击 `呼叫 TA 的 AI 分身先聊聊`；
- 从我的样本库点击 `继续问 AI 分身`。

聊天页左侧展示：

- 对应前人的头像卡；
- 前人昵称；
- 路径信息；
- 说明：AI 分身基于 TA 的公开内容生成，不代表本人实时观点。

聊天消息：

- 用户问题来自提问面板输入框；
- 后端接口可用时走 `/api/personas/chat`；
- 接口失败时使用本地 mock 回复兜底。

## 9. 我的样本库

### 9.1 初始状态

初始状态：

```js
state.savedQuestionGroups = [];
state.hasSubmittedQuestion = false;
state.submittedQuestion = "";
```

用户刚进入首页、未完成搜索、未保存样本时，点击 `我的样本库` 显示空状态。

不会因为以下原因生成样本库内容：

- 首页有默认底纹词；
- mock people 数据存在；
- 用户进入首页；
- 用户进入路径页；
- 用户没有点击 `存入我的样本库`。

### 9.2 空状态

空状态文案：

```text
你还没有保存样本
当你遇到一个想继续参考的人，可以把 TA 收进这里。
```

按钮：

```text
去寻找走过的人
```

点击后回到首页。

### 9.3 按问题分组

保存样本后，样本库按用户已提交的问题分组。

结构：

```text
我的样本库

“问题 A”  已保存 2 个样本
  样本 1
  样本 2

“问题 B”  已保存 3 个样本
  样本 1
  样本 2
  样本 3
```

数据结构：

```js
state.savedQuestionGroups = [
  {
    id: "q_xxx",
    question: "不工作了之后，我想去新西兰生活",
    createdAt: 1715673600000,
    lastSavedAt: 1715677200000,
    samples: [
      { personId: "yu", savedAt: 1715677200000 }
    ]
  }
];
```

排序：

- 问题组按 `lastSavedAt DESC`，再按 `createdAt DESC`；
- 组内样本按 `savedAt DESC`。

### 9.4 样本库视觉

当前视觉原则：

```text
问题组是卡片，样本是列表
```

问题组：

- 外层白色卡片；
- 标题显示问题文本；
- `已保存 x 个样本` 做成弱标签；
- 长问题单行省略。

样本：

- 不再是卡片里套卡片；
- 改成无边框列表行；
- 样本之间用浅分割线；
- hover 不出现特殊底色；
- 左侧头像；
- 中间昵称、路径、概要；
- 右侧两个边框按钮。

样本行按钮：

```text
回看 TA 的经历
继续问 AI 分身
```

## 10. 知乎圈子出口与草稿页

```text
想问问更多人？
```

要求：

- 不放进人物样本卡；
- 不是核心动作；
- 按钮使用弱边框，不抢 `向 TA 提问`。

点击后进入草稿页。

草稿页只保留：

- 标题输入框，label：`标题`
- 正文输入框，label：`正文`
- 按钮：`发布`
- 按钮：`返回样本库`

已删除：

- 推荐发布圈子；
- 说明文案；
- `再改改` 按钮；
- `去知乎圈子发布` 文案。

## 11. 视觉系统

整体保持 Intercom 暖米色 / 浅奶油色方向。

原则：

- 背景干净，不使用明显网格、点阵、黑客地图、旧纸张纹理；
- 主按钮使用暖黄色渐变；
- 弱按钮使用浅黄色边框或弱文字；
- 卡片阴影轻；
- 大量阅读区域使用浅色纸面；
- 不恢复暗黑科技风；
- 不恢复拓扑图。

当前核心颜色方向：

```css
--bg: #FBF7EF;
--bg-2: #F7F1E7;
--panel: rgba(255, 255, 255, 0.82);
--paper: #FFFDF8;
--text: #26231F;
--muted: #6F685F;
--accent-warm: #D89A45;
--accent-warm-soft: #FFF1C7;
```

## 13. 对后端同学的合并提醒

这版主要是前端交互和 UI 结构更新，原则上不要求新增后端接口字段。

但如果后端同学已经基于旧版 HTML 做过本地联调修改，请不要直接全量覆盖他那边的文件。建议：

1. 先确认他是否改过前端 HTML；

3. 如果改过，请让他提供当前 HTML 或 diff；
4. 再按模块合并：
   - 首页居中；
   - 右侧沉浸式提问面板；
   - 样本库按问题分组；
   - 路径卡内部排版；
   - 最新弱化视觉样式。
