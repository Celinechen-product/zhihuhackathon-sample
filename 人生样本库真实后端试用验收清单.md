# 《人生样本库》真实后端试用验收清单 V1.0

> 适用阶段：后端最小闭环跑通后，用真实知乎搜索结果测试产品是否成立。  
> 核心目标：不是验证“页面能不能显示”，而是验证产品能不能稳定做到——**从知乎真实公开内容里，找到走过相似困境的人，并结构化为可信的前人路径和人物样本卡。**

---

## 0. 验收前提

在开始真实试用前，需要确认后端至少跑通以下链路：

```text
用户输入 query
→ 可选轻量追问
→ 问题理解
→ 生成知乎搜索关键词
→ 调用知乎搜索 API
→ 获取真实知乎内容
→ 筛选真实经历
→ 抽取人物信息
→ 聚类生成前人路径
→ 生成人物样本卡
→ 返回前端渲染
```

如果这条链路没有跑通，不进入产品验收，只做接口联调。

---

## 1. 验收总原则

### 1.1 第一优先级：真实性

任何结果只要无法证明来自知乎真实公开内容，就不应该进入「前人路径」和「人物样本卡」。

必须满足：

- 有 `source.url`
- 能追溯到知乎问题 / 回答 / 文章 / 想法
- 人物经历来自原文，不是 AI 推断补全
- 时间线、转折点、结果、代价必须有原文依据

### 1.2 第二优先级：路径可信

路径必须像是从真实内容里聚类出来的，而不是 AI 先拍脑袋生成的人生建议。

合格路径应该像：

```text
先裸辞休息一阵，再重新找工作
从大城市回老家，降低生活成本
继续留在原城市，但换低压工作
先考研 / 考公，换一条更稳定路径
```

不合格路径像：

```text
勇敢追梦型
理性规划型
突破自我型
拥抱变化型
寻找人生意义型
```

这些太像咨询话术，不像真实人群路径。

### 1.3 第三优先级：样本卡可信

人物样本卡不追求完整，而追求真实。

允许：

- 只有一段经历摘要；
- 没有完整时间线；
- 只有关键转折点；
- 只有一个原文片段能说明 TA 的处境。

不允许：

- 原文没有时间线，AI 编出完整时间线；
- 原文只是观点，包装成个人经历；
- 原文没有结果，AI 编出“后来怎样”；
- 原文没有明确代价，AI 编出“代价”。

### 1.4 第四优先级：不替用户做决定

产品输出应该始终是：

```text
这些人曾经这样走过，你可以参考他们的经历。
```

而不是：

```text
你应该这样做。
```

## 2. 试用测试问题集

测试问题来自用户会真实提出的“人生相关问题”，并结合当前产品典型场景做了改造。建议第一轮至少测试 12 个问题，覆盖职业、家庭、关系、迁移、年龄焦虑、教育选择等类型。

### 2.1 P0 必测问题

| 序号 | 类型 | 测试 query | 为什么要测 |
|---|---|---|---|
| 1 | 裸辞 / 职业重启 | 30岁裸辞后，我现在找不到合适的工作，该怎么办？ | 核心高频场景，能检验裸辞、失业、转行、降薪等路径是否能分开 |
| 2 | 迁移 / 生活方式 | 不工作之后，我想去新西兰生活，怎么判断自己是真的适合，还是只是想逃离？ | 当前 Demo 主场景，验证轻量追问和路径聚类 |
| 3 | 大城市 / 回老家 | 30岁单身女，继续沪漂还是回老家？ | 检验城市选择类问题能否聚类出真实路径 |
| 4 | 工作 / 身心压力 | 工作干到怀疑人生，可又不敢辞职，怎么办？ | 检验是否能识别“职业倦怠”而不是简单给建议 |
| 5 | 失业 / 现金流 | 30岁失业一年，找不到工作，该怎么办？ | 检验结果是否能覆盖降薪、小公司、副业、转行、考编等真实路径 |
| 6 | 转行 | 30岁转行，真的还来得及吗？ | 检验路径是否来自真实转行案例，而不是泛泛励志 |
| 7 | 家庭 / 边界 | 30岁这年，我终于想和控制欲强的父母划清边界，应该怎么做？ | 检验家庭关系类是否能找到真实经历，而不是心理咨询化 |
| 8 | 婚恋 / 相亲 | 30岁还没结婚的人，面对相亲压力该怎么办？ | 检验亲密关系和家庭期待是否能抽到真实样本 |
| 9 | 教育 / 读研 | 三十岁想开启新的人生，应该选择读研还是转行到新行业？ | 检验教育选择和职业选择混合场景 |
| 10 | 留学 / 人生重启 | 人到中年裸辞留学，真的能重启人生吗？ | 检验留学内容是否能区分经历、攻略、观点 |
| 11 | 赚钱 / 责任 | 25岁一事无成，什么也不会，还能靠什么赚钱？ | 检验低技能、赚钱焦虑类问题是否能找到真实路径 |
| 12 | 年龄焦虑 | 30岁还没什么成就，算不算人生失败？ | 检验系统能否避免鸡汤化、咨询化 |

### 2.2 P1 扩展测试问题

| 类型 | 测试 query |
|---|---|
| 体制内 / 离职 | 30岁县城公务员被职场霸凌，五年服务期满后该不该辞职？ |
| 女性成长 | 40岁以上女性会给30岁女性什么人生建议？ |
| 考研 / 上岸 | 25岁要不要继续考研或者去考公？ |
| 毕业迷茫 | 20岁刚毕业，我的出路在哪里？ |
| 护理职业 | 25岁护理本科毕业，没有工作经验，到大城市怎么找工作？ |
| 稳定 vs 冒险 | 25岁，存款8万，纠结要不要裸辞回老家？ |
| 创业 / 合伙 | 30岁夫妻创业两年，现在有人拉我们合并，要不要放弃已有客户资源？ |
| 独身养老 | 30岁的独身主义者，怎么安排养老计划？ |
| 职业技能 | 20岁毕业后，如何培养一项脱离熟人社会也能生存的技能？ |
| 人生虚无 | 30岁后，感觉人生很虚无该怎么办？ |

## 4. 分项评分标准

每个 query 建议按 100 分评分，不是为了追求数字精确，而是为了快速定位问题出在哪。

| 模块 | 权重 | 合格标准 |
|---|---:|---|
| 问题理解 | 10 | 能识别用户真正困惑，不把问题理解偏 |
| 轻量追问 | 10 | 只在必要时追问，追问能提升检索变量 |
| 搜索关键词 | 15 | 能覆盖知乎真实内容可能出现的表达，不只复述 query |
| 真实经历筛选 | 20 | 能过滤纯观点、鸡汤、营销、无经历内容 |
| 路径聚类 | 15 | 路径像从真实经历里归纳出来，而不是先验建议 |
| 人物样本卡 | 15 | 人物有真实经历，字段不硬编，结构清晰 |
| 原文溯源 | 10 | 每个人物和关键经历都有完整 source.url |
| 文案可信度 | 5 | oneLine / lesson / path desc 不鸡汤、不 AI 味 |

### 4.1 评分解释

```text
85-100：可进入 Demo 演示素材池
70-84：方向可用，但需要提示词或筛选规则优化
50-69：勉强跑通，但产品可信度不足
低于 50：本轮结果不可用，需要回到检索 / 筛选 / 抽取环节修
```

## 5. 总体验收标准

### 5.1 P0 通过标准

第一轮至少测试 12 个 P0 query。

产品可以进入 Demo 演示前，必须满足：

```text
≥ 8 个 query 得分 ≥ 70
≥ 4 个 query 得分 ≥ 85
0 个 query 出现严重编造人物经历
0 个 query 出现 source.url 缺失但仍展示为样本卡
0 个 query 出现明显“AI 替用户做决定”
```

如果达不到，不要急着优化页面，先修后端检索、筛选、抽取和提示词。

### 5.2 单个 query 最低可用标准

每个 query 至少应返回：

```text
2-4 条可信前人路径
每条路径 1-3 个真实样本
每个样本至少 1 个 source.url
每个样本至少 1 段能证明 TA 真实经历的原文依据
```

允许低召回时降级为：

```text
只返回 1 条路径
只返回 1-2 个样本
明确告诉用户：目前只找到这些较接近的公开经历
```

不允许为了凑满 3-5 条路径而牺牲真实性。

## 6. 轻量追问验收标准

### 6.1 什么时候应该触发追问

应该触发追问的 query：

```text
30岁以后怎么办？
我现在很迷茫，该怎么办？
不工作之后，我想去新西兰生活
30岁还没成就，算失败吗？
```

原因：这些问题缺少关键变量，直接搜索容易泛化。

### 6.2 什么时候不应该触发追问

不应该追问的 query：

```text
30岁失业一年，找不到工作，该怎么办？
25岁存款8万，纠结要不要裸辞回老家？
30岁县城公务员被职场霸凌，五年服务期满后该不该辞职？
三十岁想开启新的人生，应该选择读研还是转行到新行业？
```

原因：这些问题已经有足够明确的场景变量，可以直接搜索。

### 6.3 追问质量标准

合格追问应该是：

```text
为了找到更接近的真实经历，想再确认一句：
你现在更接近哪种情况？

[只是想暂停一阵子]
[认真考虑长期留下]
[还没想清楚]
[跳过，先帮我找找看]
```

不合格追问：

```text
请描述你的家庭、收入、职业、性格和人生目标
AI 需要更多信息才能判断你的真实需求
你是否已经做好长期移民准备？
```

问题在于太重、太像诊断、太像咨询问卷。

### 6.4 追问结果验收

追问不能只是 UI 装饰，必须真的进入搜索策略。

检查方式：

```text
同一个 query：
A. 不选择追问，直接搜索
B. 选择一个追问选项后搜索

观察：
- 搜索关键词是否变化
- paths 是否更聚焦
- people 是否更接近用户意图
```

如果选择追问后结果没有任何变化，说明追问模块没有实际产品价值。

## 7. 路径 paths 验收标准

### 7.1 合格路径特征

每条路径应该能回答：

```text
这一组人为什么被放在一起？
他们的共同处境是什么？
他们做了什么相似选择？
这个选择带来了什么结果或代价？
```

合格字段：

```js
{
  id: "return_home",
  name: "裸辞后回老家重新开始",
  count: 3,
  desc: "这些人没有继续在大城市硬撑，而是先回到低成本环境里重新找工作和生活节奏。"
}
```

### 7.2 不合格路径特征

不合格路径通常有这些问题：

- 像建议，不像路径；
- 抽象词太多；
- 没有人群归类感；
- 和人物样本没有对应关系；
- 只是为了凑满 3-5 条。

不合格示例：

```text
积极面对型
寻找自我型
顺其自然型
理性决策型
勇敢突破型
```

### 7.3 路径数量规则

| 真实召回情况 | 展示策略 |
|---|---|
| ≥ 3 类明显不同路径 | 展示 3-5 张路径卡 |
| 只有 2 类路径 | 展示 2 张路径卡，不硬凑 |
| 只有 1 类路径，但 ≥ 2 人 | 可直接进入该路径样本库，或只展示 1 张路径 |
| 只有 1 个有效人物 | 不展示路径选择，直接进入人物样本卡 |
| 没有有效人物 | 展示低召回提示，不生成假路径 |

## 8. 人物 people 验收标准

### 8.1 合格人物必须满足

每个 `person` 至少满足：

```text
有知乎昵称或可识别作者名
有 source.url
有一段原文能证明 TA 真的经历过类似问题
有明确 sample_type
没有超出原文的信息编造
```

建议结构：

```js
{
  id: "...",
  name: "...",
  sample_type: "full_experience | fragmented_experience | opinion_with_experience",
  path_id: "...",
  oneLine: "...",
  source: {
    title: "...",
    url: "https://www.zhihu.com/..."
  }
}
```

### 8.2 三类 sample_type 验收

#### A. 完整经历型：`full_experience`

适用：

- 原文有明确起点；
- 有过程；
- 有选择；
- 有结果；
- 可抽出时间线。

可以展示：

```text
TA 是谁
为什么匹配到 TA
TA 的经历时间线
TA 的真实经历
```

#### B. 碎片经历型：`fragmented_experience`

适用：

- 原文只有部分经历；
- 有处境和某个转折，但没有完整结果；
- 不能硬抽完整时间线。

展示策略：

```text
TA 是谁
为什么匹配到 TA
TA 提到的关键片段
TA 的真实经历
```

不要展示完整时间线。

#### C. 观点中带个人经历型：`opinion_with_experience`

适用：

- 回答主要是观点；
- 但其中有一段自己的真实经历；
- 可作为辅助样本，但不应作为强样本。

展示策略：

```text
TA 的相关经历片段
TA 的观点来自什么经历
TA 的真实经历
```

需要弱化人物卡权重，避免包装成完整人生故事。

### 8.3 必须丢弃的人物

以下结果不能进入人物样本卡：

```text
纯观点，没有个人经历
营销引流
鸡汤文
泛泛建议
只有一句话情绪宣泄
没有 source.url
无法判断作者是否亲历
内容与 query 只有关键词重合，但处境不相似
```

## 9. timeline 验收标准

### 9.1 可以生成 timeline 的条件

只有当原文明确包含时间、阶段或顺序时，才生成 timeline。

可用依据包括：

```text
“我25岁时……”
“毕业后第一份工作……”
“裸辞三个月后……”
“后来回老家……”
“2022年……”
```

### 9.2 不可以生成 timeline 的情况

如果原文只是这样：

```text
我也迷茫过，后来慢慢好了。
```

不能生成：

```text
2020：陷入迷茫
2021：重新开始
2022：找到方向
```

这是典型 AI 编造。

### 9.3 timeline 降级展示

如果没有足够依据，改成：

```text
TA 提到的关键经历
```

结构：

```text
- 曾经遇到的处境：...
- 做过的尝试：...
- 原文里明确提到的结果：...
```

没有结果就写：

```text
原文未明确提到后续结果
```

## 10. 文案可信度验收标准

### 10.1 oneLine

合格：

```text
她裸辞后回到老家住了半年，发现真正缓解压力的不是离开大城市，而是先把现金流和作息稳定下来。
```

不合格：

```text
她勇敢追寻内心，终于找到了属于自己的光。
```

### 10.2 lesson

建议把 `lesson` 改成更克制的字段名：

```text
TA 这段经历里可以参考的点
```

或：

```text
这段经历提醒了什么
```

不建议叫：

```text
关键经验
人生启示
TA 留下的答案
```

### 10.3 path desc

合格：

```text
这些人没有马上转行，而是先找了一份低压工作，让收入和状态恢复稳定。
```

不合格：

```text
他们用勇气和智慧走出了人生低谷。
```

## 11. source.url 验收标准

每个样本必须有：

```js
source: {
  title: string,
  url: string,
  type: "answer | article | question | pin | unknown"
}
```

最低要求：

- `url` 不能为空；
- URL 应该是知乎域名；
- 前端点击可打开；
- 原文阅读器必须显示「查看知乎原文」；
- 如果只有问题链接，没有回答链接，需要标记为低可信或待补全。

### 11.1 source 缺失处理

如果 `source.url` 缺失：

```text
不能展示为正式人物样本卡
只能进入日志 / debug
或作为低可信候选，不给用户看
```

## 12. 与「结果评估与提示词调优」助手的协作方式

你新增的评估助手适合做单次结果深度质检。建议配合方式如下：

### 12.1 本清单负责什么

本清单负责：

```text
定义试用问题集
定义通过标准
定义红线
定义评分维度
定义整体验收节奏
```

### 12.2 结果评估助手负责什么

每次后端返回一组结果后，把以下内容发给它：

```md
## 用户 query
...

## 问题理解结果
...

## 搜索关键词
...

## paths
...

## people
...

## 原始知乎搜索结果 / 日志（可选）
...
```

它负责输出：

```text
总体结论
分项评分
主要问题
应保留结果
应丢弃 / 降级结果
提示词调优建议
可直接发给 Codex 的修改指令
```

### 12.3 两者怎么衔接

建议流程：

```text
你用本清单挑选测试 query
→ 跑真实后端
→ 把单次结果发给结果评估助手
→ 拿到质检结论和提示词建议
→ 同步给“实操搭建后端”对话
→ 由它翻译成 Codex 指令
→ Codex 修改
→ 再用本清单复测
```

## 13. Demo 演示前红线清单

Demo 前必须逐条检查。

### 13.1 绝对红线

出现任意一条，都不建议演示该 query：

```text
人物经历明显是 AI 编的
timeline 没有原文依据
source.url 缺失
路径名像鸡汤建议
人物只是观点作者，没有亲历内容
AI 分身说得像本人正在实时回答
为了凑满路径和人物，塞入明显不相关结果
```

### 13.2 高风险但可降级

以下情况不一定要删除，但必须降级展示：

```text
只有碎片经历，没有完整时间线
只有观点中夹带个人经历
只有问题页链接，没有具体回答链接
人物经历相关，但和 query 不完全匹配
路径只有 1 个人支撑
```

降级方式：

```text
少展示
弱化权重
不生成完整时间线
不放在第一屏
标注“这是一段相关经历片段”
```

## 14. 一轮真实验收的推荐节奏

### 第 1 轮：跑通链路

目标：

```text
确认真实知乎内容能进入前端
```

只测 3 个 query：

```text
30岁裸辞后，我现在找不到合适的工作，该怎么办？
30岁单身女，继续沪漂还是回老家？
工作干到怀疑人生，可又不敢辞职，怎么办？
```

通过标准：

```text
能返回真实 source.url
能生成至少 1 条可信路径
能生成至少 1 个可信人物样本
```

### 第 2 轮：测泛化

目标：

```text
确认不同人生问题都能被结构化
```

测试 P0 12 个 query。

通过标准：

```text
≥ 8 个 query 得分 ≥ 70
```

### 第 3 轮：测演示稳定性

目标：

```text
选出 Demo 最稳的 2-3 个 query
```

标准：

```text
结果可信
路径好理解
样本有真实原文
人物卡不假
演示链路顺畅
```

建议最终 Demo query 不超过 3 个。

## 15. 最终 Demo 素材池选择标准

一个 query 可以进入 Demo 素材池，需要同时满足：

```text
1. 用户问题真实、有代入感
2. 轻量追问可有可无，但不能卡住
3. 返回路径 2-4 条即可，不必硬满 5 条
4. 每条路径名中性、具体、像真实人群
5. 至少有 3 个可信人物样本
6. 至少有 3 个可打开的知乎原文链接
7. 样本卡没有明显编造
8. 文案不鸡汤
9. 能一句话讲清楚产品价值
```

推荐 Demo 讲法：

```text
我们不是让 AI 告诉用户该怎么选。
我们做的是把知乎里走过相似困境的人找出来，
整理成几条真实前人路径，
让用户先看见别人怎么走、遇到了什么、付出了什么代价，
再自己判断。
```

## 16. 当前阶段最重要的判断

后端刚搭好时，不要急着问：

```text
页面漂不漂亮？
AI 分身好不好聊？
样本库能不能长期保存？
```

应该先问：

```text
这些人真实吗？
这些经历真实吗？
路径像真实聚类吗？
原文能追溯吗？
有没有为了凑结果而编？
```

只要这四个问题不过关，产品就还没有成立。
